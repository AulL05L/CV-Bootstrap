# cv-bootstrap

Personal CV website — **Aulia Destiana**

Dark-mode, single-page CV + portfolio built with Bootstrap 5 (via npm),
locally-hosted Google Fonts (via [@fontsource](https://fontsource.org/)),
Font Awesome 6.5 (CDN), and vanilla JavaScript. No build framework required.

---

## Folder structure

```
cv-bootstrap/
├── index.html              ← main page (all sections)
├── package.json            ← npm deps: bootstrap, @fontsource/syne, @fontsource/outfit
├── build.js                ← copies vendor files into assets/
├── assets/
│   ├── css/
│   │   └── style.css       ← all custom styles + @import for local fonts
│   ├── fonts/              ← generated by `npm run build`
│   │   ├── syne/
│   │   └── outfit/
│   ├── img/
│   │   └── hero.jpg        ← replace with your own photo
│   ├── js/
│   │   └── main.js         ← smooth scroll, nav close, scroll reveal
│   └── vendor/             ← generated by `npm run build`
│       ├── css/bootstrap.min.css
│       └── js/bootstrap.bundle.min.js
└── README.md
```

## How to run locally

### Prerequisites
- [Node.js](https://nodejs.org/) 18+ installed

### Steps

```bash
# 1. Enter the project folder
cd cv-bootstrap

# 2. Install Bootstrap + fontsource packages
npm install

# 3. Copy vendor/font files into assets/
npm run build

# 4a. Open directly in browser
#     Double-click index.html  — OR —

# 4b. Use the built-in dev server (auto-reload)
npm start
#     Opens http://localhost:5500
```

> **Important:** `npm run build` must be run at least once before opening
> `index.html`. Otherwise Bootstrap and the local fonts won't load.

## How to deploy to GitHub Pages

1. Push the entire project folder to a GitHub repository:
   ```bash
   git init
   git add .
   git commit -m "initial commit"
   git remote add origin https://github.com/<username>/<repo>.git
   git push -u origin main
   ```

2. In your repository go to **Settings → Pages**.

3. Under *Source* select **Deploy from a branch**, choose `main` / `root`.

4. Click **Save**. After ~1 minute your site is live at
   `https://<username>.github.io/<repo>/`.

> **Tip:** Commit the generated `assets/vendor/` and `assets/fonts/` folders
> (remove those lines from `.gitignore` first), so GitHub Pages can serve
> the files without needing a build step on the server.

## How to replace the hero photo

1. Prepare a square image of yourself (recommended: **480 × 480 px** or larger).
2. Save it as `assets/img/hero.jpg`.
3. Refresh your browser — done.

The `<img>` tag has an `onerror` fallback that shows a generated avatar
(`ui-avatars.com`) if the file is missing, so the site never breaks.

## Customisation quick-reference

| What to change | Where |
|---|---|
| Name / subtitle / bio | `index.html` — Hero section |
| School names in timeline | `index.html` — Education section |
| Skill percentages | `index.html` `data-width="…"` + displayed % |
| Social media links | `index.html` — Hero + Contact sections |
| Color palette | `assets/css/style.css` — `:root` variables |
| Fonts | `package.json` deps + `@import` in `style.css` |
